本文档用于备份逑贝互联网医院在彻底切断第三方外包商系统、走向 100% 自研独立部署过程中的**核心资产追讨清单**及**自持运营硬性预算明细**。

## 一、 资产追讨与金蝉脱壳准备清单（需向旧外包商索要）

在正式通知外包商解约前，必须通过商务、法律或合规手段，将属于医院的以下核心数字资产完好无损地追回来：

### 1.1 医疗核心数据资产
* [ ] **历史患者与就诊人数据库（SQL脚本或Excel）：** 包含所有已注册患者的姓名、身份证号（解密明细）、手机号、微信 `OpenID/UnionID`、历史挂号记录。
* [ ] **电子病历与历史处方数据归档：** 过去所有医生书写的电子病历文本、开具的 Rp 处方药明细、处方编号、诊断 ICD-10 编码。这是卫健委抽查必看数据，必须追回。
* [ ] **药品字典数据：** 现有线上院内药房的所有药品名称、规格、剂型、批准文号、生产厂家、用法用量库。可以直接导入新系统，免去重新录入几百款药品的苦力活。

### 1.2 外部合规网关账号与商务主体（极其重要）
* [ ] **天津卫健委监管平台接入凭证：** 确认监管平台数据上报的测试环境与生产环境的 `AppKey`、`AppSecret`、加密通道证书等是否在公司自己手中。
* [ ] **CA 电子签名机构合同与账号：** 明确当前系统使用的数字证书是哪家 CA 机构颁发的。如果是外包商代签的，需向其要到技术对接文档，或者准备在新系统中直接重新签约。
* [ ] **微信支付商户号控制权：** 登录微信商户平台，确认商户号的管理员是上海逑贝或天津逑贝的员工，确认 APIv3 密钥及退款 API 证书在自己手里。

## 三、 新旧系统切换技术路径演练（防惊动外包商）

为了确保医院目前的线上营业不受影响，自研新系统上线时必须采取“暗度陈仓”的平滑切换技术：

1. **并行开发：** 研发团队使用新注册的小程序 AppID，配合自研的 Python FastAPI 后端，在自己的阿里云服务器上进行闭环联调，**期间旧系统照常运行，不向外包商透露任何风声**。
2. **合规通关：** 提前完成新系统的“等保三级测评”，并联系天津卫健委信息中心，用新系统的后台代码完成数据上报接口的测试与联调验收。
3. **一键切流（切断旧生态）：** 在深夜患者最少的时间窗口，登录上海母公司的微信公众号后台，将自定义菜单、推文、官网等所有流量入口对应的**旧小程序 AppID 瞬间更改为新小程序的 AppID**。
4. **正式止损：** 观察 48 小时，确认新小程序的挂号、视频、开方缴费以及资金流水全部正常进账后，正式向旧外包商发出解约函，彻底废弃旧系统。
