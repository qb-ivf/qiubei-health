# 全链路真机走查清单（上线前集成验证）

医生端小程序审核通过后，用**两台真机**（患者 / 医生）+ 运营后台，把核心业务跑一遍。
后端：`https://api.qb-medical.cn`（生产）。建议先用 1 分钱号源测真实支付，走完改回正常价。

## 前置确认
- [ ] 患者端、医生端小程序均为**最新体验版**，`app.js` baseUrl = `https://api.qb-medical.cn`
- [ ] 两个小程序后台均配 request 合法域名 `https://api.qb-medical.cn`、socket `wss://api.qb-medical.cn`
- [ ] 服务器 `.env` 已配 `WX_PATIENT_*`/`WX_DOCTOR_*`/`WX_PAY_NOTIFY_URL`/`TRTC_*`，并 `restart api`
- [ ] 视频要真实：小程序「实时音视频」类目已过 + 官方 `trtc-wx` SDK 已覆盖占位桩
- [ ] 准备一个 1 分钱挂号费的医生号源（测完改回）

## 一、登录与实名
- [ ] 患者端微信登录成功；`getPhoneNumber` 拿到**真实手机号**（后端日志无 getuserphonenumber 失败）
- [ ] 患者添加就诊人 + 实名（身份证格式校验）
- [ ] 医生端微信登录；**未审医生**进入资质提交页（非接诊大厅）

## 二、医生资质审核闭环
- [ ] 医生提交执业资质（姓名/资格证/执业证）→ 状态变「审核中」
- [ ] 运营后台「医生资质终审」看到该医生 → 通过
- [ ] 医生重新进入 → 直达接诊大厅；未通过前访问接诊接口应被拒（403）

## 三、挂号 + 真实支付
- [ ] 患者选医生 → 选时段 → 显示挂号费（用 1 分测试号 = ¥0.01）
- [ ] 点支付 → 弹**微信收银台** → 真实付款
- [ ] 后端日志出现 `/orders/pay/callback` 且验签通过；订单 PENDING→WAITING
- [ ] 商户平台能查到这笔交易
- [ ] 候诊队列：该订单只出现在**对应医生**的队列里（别的医生看不到）

## 四、视频问诊
- [ ] 医生在接诊大厅点「接诊」→ 患者端收到 CALL_INVITE，进入视频房
- [ ] 双向音视频画面正常；麦克风/摄像头开关、前后摄像头切换、扬声器/听筒
- [ ] 网络波动/退后台恢复（可选）

## 五、开方 + 药师审方
- [ ] 医生写病历、开药、`数字签名并发送处方` → 患者端收到 CALL_FINISHED → 跳处方页
- [ ] 运营后台「药师审方」看到待审处方 → 通过 / 驳回
- [ ] 处方 PDF 生成（CA 签章目前占位，确认 PDF 能出）

## 六、药费支付 + 物流
- [ ] 处方通过后，患者端药费支付（真实微信支付，out_trade_no 带 `-DRUG` 后缀）
- [ ] 支付成功 → 订单 PRESCRIBED→FINISHED + 分账记账
- [ ] 物流时间轴展示

## 七、退款 / 通知（抽查）
- [ ] 候诊中订单申请退款 → REFUNDED + 回补号源
- [ ] 关键节点订阅消息（目前下发占位，确认不报错）

## 八、运营后台（RBAC）
- [ ] admin / pharmacist / finance 各用对应账号登录，菜单可用、越权接口返回 403
- [ ] 财务对账提现、监管上报面板能拉到数据
