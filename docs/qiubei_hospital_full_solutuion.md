# 逑贝互联网医院（双端小程序+PC端+服务端）全链路合规方案与技术架构 PRD

## 一、 全局产品矩阵与物理隔离架构

为了满足微信官方最新审核红线，并确保医疗责任主体明确，系统采用“两端独立小程序 + 一端 PC 网页 + 统一中央服务端”的商用级标准架构：

```text
       ┌──────────────────┐          ┌──────────────────┐
       │ 患者端小程序(壳) │          │ 医生端小程序(壳) │
       │  (名称: 逑贝健康) │          │(名称: 逑贝医生端)│
       └─────────┬────────┘          └─────────┬────────┘
                 │                             │
                 │   HTTPS (业务) / WSS (信令) │
                 ▼                             ▼
       ┌────────────────────────────────────────────────┘
       │  ▶ 【统一云中央服务端 (大脑 - Python FastAPI)】
       │  - 负责状态机、财务分账、数据加密、音视频 Token 计算
       └─────────────────────────┬──────────────────────┘
                                 ▲
                                 │ HTTPS (管理 API)
                                 │
                       ┌─────────┴────────┐
                       │  PC 运营管理端   │
                       │(药师审方/财务行政)│
                       └──────────────────┘

```

1. **患者端（微信小程序 A - 暂定名“逑贝健康”）：** 供患者挂号、音视频问诊、查方缴费、物流追踪。
2. **医生端（微信小程序 B - 暂定名“逑贝医生端”）：** 供大夫一键排班、切换接诊状态、视频叫号接诊。
3. **PC管理端（电脑浏览器网页 - Vue3 + Element Plus）：** 供药师在电脑大屏进行“处方双盲审核”，财务对账提现审批，行政录入医生资质。
4. **中央服务端（后端系统 - Python FastAPI）：** 隐藏在幕后的核心大脑，掌管所有数据持久化、微信支付状态机、WebSocket 媒婆调度、CA 签名与卫健委上报。

## 二、 核心业务生命周期与数据状态机

系统以“挂号支付”**为起点，以**“处方通过与药费支付”为终点。中央服务端严格执行封闭式状态机，任何前端无权擅自修改状态。

### 2.1 订单状态机定义 (Order Status)

* `0: PENDING` (待支付)
* `1: WAITING` (候诊中) -> 用户付完挂号费后，服务端通过微信支付 Webhook 幂等性更新，并将就诊卡送入 Redis 排队队列。
* `2: CONSULTING` (问诊中) -> 医生在医生端点击“立即接诊”，服务端锁定该医生，并触发双端视频连线。
* `3: AUDITING` (待药师审核处方) -> 医生在视频诊室开具处方并提交。
* `4: REJECTED` (处方被药师驳回) -> 药师认为用药不当或配伍禁忌，退回医生端要求重开。
* `5: PRESCRIBED` (已开方/待付药费) -> 药师电脑端审方通过。
* `6: FINISHED` (问诊完成) -> 患者完成药费支付或问诊安全结束。
* `7: REFUNDED` (已退款) -> 视频未接通或网络异常引发的客服退款。
* `9: CANCELLED` (已取消) -> 挂号后 15 分钟未支付，Redis 键空间通知触发超时自动取消。

## 三、 服务端（大脑）核心技术功能与边界细节

服务端基于 **Python 3.10+ / FastAPI**，采用异步（Async/Await）架构，保证低延迟、高并发。

### 3.1 核心业务 API 引擎 (Core API Engine)

* **安全收单：** 微信支付参数（Signature）及退款证书严禁硬编码。收单回调时，服务端发起 `SELECT ... FOR UPDATE` 悲观锁行锁定，防止高并发图文问诊或重复回调导致的状态机死锁。
* **JWT 角色隔离：** 采用双重 Token 校验。患者端携带由手机号生成的 `role: patient` Token；医生端必须通过后台白名单（`doctors` 表中已认证通过的手机号）比对，下发 `role: doctor` Token，其余一律拦截。

### 3.2 WebSocket 信令中心（双端“媒婆”控制）

* **连接维系：** 在 Redis 中维护一个 Hash 结构，实时存储在线医生的 Socket ID 与患者的房间号 `room:status:[room_id]`。
* **主动唤醒机制：** 医生端发送 `INIT_CALL` 信号，服务端通过长连接向患者端广播 `type: "CALL_INVITE"`。
* **弱网保护：** 维持 4 秒一次的心跳包机制。若 12 秒内无响应，启动 30 秒断线重连保护期，30 秒后仍未重连则触发系统挂断。

### 3.3 合规对接网关 (Compliance Gateway - 命门系统)

* **天津市卫健委监管平台实时上报：** * 采用 **Celery 异步分布式任务队列**。主业务在医生点击接诊/药师审方通过时，直接向 Redis Stream / RabbitMQ 投递消息。
* 异步 Worker 负责将数据打包，按照天津卫健委最新规范（通常为 **AES-128-CBC 加密** 配合动态 **MD5-Sign** 机制），实时、异步上报医生资质、问诊聊天、开药明细。
* 具备 **5 次指数退避重试机制**（5分钟、15分钟、1小时...），防止因卫健委服务器偶发崩溃导致本院业务卡死。


* **CA 机构电子签名对接：**
* 服务端对接三方国密合规 CA 机构。医生首次入驻通过人脸识别，在云端托管托管生成医生的专属私钥数字证书。
* 药师审方通过的瞬间，网关调用 CA 的云端静默加签接口，利用 **国密 SM2 算法** 对处方进行数字证书加密，利用 Python `reportlab` 库在服务器端生成标准 PDF 处方文件，加盖带有时间戳的不可篡改红章及医生手写签名。


* **医疗档案 15 年长效存储：**
* 问诊结束后的双向视频录制文件（MP4）及处方 PDF，由服务端网关自动上传至云存储。
* 强制配置云存储生命周期（Lifecycle Policy）为“归档存储（Archive）”并设置 15 年内禁止删除与修改，完全防御卫健委执法检查。

## 四、 前端双端小程序核心交互规范（10 个页面）

前端采用 **VS Code 开发 + Less 编译 + 微信开发者工具预览**。主包控制在 2MB 以内，采用模块化分包。

### 4.1 患者端小程序（7 个页面）

1. **页面 1：小程序首页（主包）** -> 提供【自定义 NavigationBar】、就诊人快速切换滑块、搜索框及高亮绿色大字号“视频问诊”金刚区入口。
2. **页面 2：医生列表页（分包A）** -> 顶部固定科室横向滚动条及未来 7 天出诊矩阵，下方为支持分页加载的医生卡片垂直流。
3. **页面 3：医生详情与挂号确认页（分包A）** -> 选择特定就诊人与精确出诊时段，点击“立即支付”唤起 `wx.requestPayment` 原生组件。支付后重定向回首页，挂载黄色置顶排队提示条。
4. **页面 4：远程视频呼叫与接听页（分包A - 自定义全屏无导航栏）** -> 毛玻璃背景 + 头像四周向外扩散的 CSS 涟漪。WebSocket 接收到呼叫信令后，前端无条件强行路由压栈拉起本页并响铃。
5. **页面 5：全屏双向视频诊室页（分包A）** -> 原生组件 `<live-player>` 全屏铺满（拉流看医生），`<live-pusher>` 右上角圆角悬浮（推流看自己）。支持点击小窗对调画面 Z-Index 与尺寸。
6. **页面 6：诊后电子处方与药费支付页（分包A）** -> 视频挂断后强制渲染此页。展示带有 CA 红色电子公章的 Rp 处方卡片，底部常驻“去支付药费（送药上门）”大按钮。
7. **页面 7：个人中心/“我的”页面（主包）** -> 健康档案、电子病历查询、预约记录、我的处方库入口。

### 4.2 医生端小程序（3 个页面）

1. **页面 D1：医生首页/接诊大厅（主包）** -> 顶部核心常驻异步 `wx-switch` 按钮。当拨动到 **ON** 时，今日候诊室列表高亮激活，排在首位的患者卡片露出蓝色“立即接诊”按钮，点击杀入诊室。
2. **页面 D2：视频问诊与开处方一体化页面（分包B）** -> **上部固定 40vh 视频通话区，下部固定 60vh 施工作业区**。作业区包含 `[患者主诉]`、`[书写病历]`、`[开具处方]` 三项 Tab 切换。
* **硬核防遮挡技术：** 所有的 `textarea` 输入框硬性配置 `adjust-position="false"` 与 `cursor-spacing="20"`。前端监听 `bindfocus` 获取键盘真实高度，动态给下部区域增加 `padding-bottom`，**确保弹起软键盘时，上半部 40vh 的视频区雷打不动**。


3. **页面 D3：个人中心/“我的工作台”（主包）** -> 医生个人执业资产管理。包含提现中心（对接微信商家转账到零钱接口）、出诊排班日历、常用快捷回复语管理。
